# Conversational Agent OS Chat Density Design

## Context

`/conversational-agent-os` 已经具备真实线程、消息、卡片和跨线程摘要流转，但当前桌面端存在两个明显问题：

1. 左侧线程栏承担了过多正文信息，和主舞台抢注意力。
2. 中间主舞台更像“会话页 + 信息面板”，不像桌面聊天工具，导致用户难以快速聚焦。

用户要求：

- 左侧线程栏瘦身
- 右侧整体更像 PC 版微信聊天框
- 仍保留顶部的 `Top 任务卡 loop`

## Approaches Considered

### 1. 只删左侧文案

- 优点：改动最小
- 缺点：主舞台结构仍然偏“面板式工作台”，不会真正变成聊天体验

### 2. 左侧瘦身 + 中间聊天化（推荐）

- 左侧只保留线程识别和状态导航
- 中间改成“置顶任务卡 + 聊天消息流 + 底部输入框”
- 卡片详情降为轻量侧栏/下钻区，而不是和消息流平权的大面板
- 能同时保留任务优先级和聊天沉浸感

### 3. 完全照搬微信双栏

- 优点：聊天心智最强
- 缺点：会牺牲 Agent-first 的任务卡和证据下钻语义，容易退化成 IM 壳

## Chosen Direction

采用方案 2：`左侧瘦身 + 中间聊天化`。

这轮不改对象模型和 API，不动线程、消息、卡片、handoff 的持久化协议，只做信息架构和视觉优先级重排。

## Information Architecture Changes

### Left Rail

左侧线程栏只回答两个问题：

- 我现在要去哪一个线程
- 哪个线程刚刚有新变化

因此每张线程项只保留：

- 线程名
- 角色标签
- 未读标记
- 单行最新状态
- 时间

去掉：

- 线程说明正文
- `当前最重要`
- 多行最新消息摘要

### Main Stage

主舞台改成桌面微信式聊天布局：

- 顶部：当前线程栏头，显示线程名、当前人、状态
- 次顶部：`Pinned Top Task`，作为置顶任务卡
- 中部：消息流，气泡化、左右/深浅分层
- 底部：固定输入区

### Detail Layer

卡片详情不再与消息流并列成大面板，而是改成更轻的详情抽屉/侧板语义。

在当前实现里，为了降低改动范围，先保留详情区域，但视觉弱化并收窄为“辅助下钻层”。

## Visual Direction

目标不是复刻微信颜色，而是借用其交互秩序：

- 线程导航更轻、更短、更像列表
- 聊天气泡更明确，降低大面积面板感
- 输入区更固定，更像常驻聊天输入框
- 置顶任务卡更像“置顶事务”，不是第二个大 dashboard

具体设计原则：

- 左侧以单行信息为主
- 中间优先级：置顶任务 > 最新消息 > 历史消息 > 详情下钻
- 用留白、边线和气泡层次区分，不再依赖大段说明文

## Testing Strategy

本轮先补页面级失败测试，覆盖：

1. 左侧线程项不再渲染线程说明和 `当前最重要`
2. 左侧仍保留时间和未读信息
3. 主舞台出现聊天式线程头、置顶任务、消息流和输入区

视觉回归用现有页面测试和构建验证收口；如需要再补浏览器级 smoke。
