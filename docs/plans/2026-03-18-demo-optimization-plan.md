# Demo 优化方案：对齐 Agent 规格定义

日期：2026-03-18

## 1. 总体判断

当前 demo 有 25+ 页面路由、14 张任务卡、3 个角色线程。代码量大但与最新 Agent 规格存在明显偏差：
- **产品名**仍为 AI Revenue Management OS，应更新为 AI Revenue Management OS
- **页面过多**，CEO/主管专属视图与规格冲突
- **核心流程缺失**：会后流程（post-meeting flow）是规格的起点，但 demo 中无法走通
- **Agent 身份模糊**：当前对话 Agent 没有明确的 BPAgent / 客户 Agent / 商机 Agent 分工

## 3. 应该增加/强化的部分（不足）

### 3.1 【最高优先级】会后流程闭环（Post-Meeting Flow）

**现状**：规格定义的核心 user journey 无法在 demo 中走通。用户无法"发送会议链接 → Agent 理解 → 推送卡片 → confirm → apply"。

**需要实现**：

在 Conversational Agent OS 的一线销售线程中，预置一个可演示的会后流程：

1. **销售发送会议链接**：在对话中输入/粘贴会议链接（或选择素材类型）
2. **Agent 自动识别**：显示"正在理解会议内容…" → 生成会议摘要卡片
3. **会议摘要卡片**：
   - 关键议题与决策
   - 参会人态度判断
   - 识别的关键信号（预算、竞品、时间线等）
   - 建议的下一步
4. **销售 confirm**：确认摘要准确
5. **Agent apply**：写入客户/商机上下文 → 显示"已更新客户X的 Account Thread"
6. **商机 Agent 响应**：推送新卡片
   - 阶段推进建议（"建议从'需求确认'推进到'方案评估'"）
   - 风险变化提醒
   - 下一步行动建议
7. **销售 confirm 下一步** → apply

**实现方式**：在 seed 数据中预置这个完整流程的消息序列，让用户可以逐步体验。或者让模型调用真正生成这些卡片。

### 3.2 【高优先级】Agent 身份与触发时机

**现状**：当前 Agent 是一个统一的"AI Assistant"，没有 BPAgent / 客户 Agent / 商机 Agent 的区分。

**需要改进**：

1. **一线销售线程**：Agent 自我介绍为"你的销售业务伙伴"
2. **主管线程**：Agent 自我介绍为"你的管理业务伙伴"
3. **卡片来源标注**：每张卡片标注来自哪个 Agent
   - "来自商机 Agent" → 阶段推进、风险预警
   - "来自客户 Agent" → 健康度变化、Account Thread 更新
   - "来自你的 BPAgent" → 今日任务、执行提醒
4. **主动推送**：线程开盘时，BPAgent 主动推送"今日关注"卡片（已部分实现，需要强化内容）

### 3.3 【高优先级】日常任务推送（每日开盘）

**现状**：seed 中有"今日关注 3 件事"但内容是静态的。

**需要改进**：

- 今日关注卡片应该明确标注来源：
  - "商机① Agent 提醒：停滞 14 天，建议跟进"
  - "客户X Agent 提醒：上次会议后未回复，关系降温"
  - "会议准备：下午 2 点和客户Y 开会"
- 每张卡片可展开查看证据
- 每张卡片可直接 confirm 执行

### 3.4 【中优先级】素材上传整合进对话

**现状**：素材上传（intake）是独立页面 `/intake`，与主对话界面断裂。

**需要改进**：

- 在 Conversational Agent OS 的 Composer 区域，素材上传后应该直接在对话中触发 Agent 理解流程
- 不再需要跳转到 `/intake`
- 上传流程：选择素材类型 → 输入/粘贴内容 → Agent 开始处理 → 在对话中返回结构化结果

### 3.5 【中优先级】向上求助/上报流程

**现状**：销售到主管的上报只在 task card 系统中有，但在主对话中没有体现。

**需要实现**：

- 销售在对话中说"这个折扣我批不了，需要主管帮忙"
- BPAgent 生成上报卡片，标注"待主管审批"
- 在主管线程中自动出现审批卡片
- 主管 confirm/reject → 结果回传到销售线程

### 3.6 【中优先级】会议前准备卡片

**现状**：未实现。

**需要实现**：

- 基于日历检测，在会议前 30 分钟推送准备卡片
- 卡片内容：上次会议遗留问题、客户最新动态、商机当前状态、建议讨论要点
- Demo 中可以预置一张这样的卡片在 seed 里

### 3.7 【低优先级】产品名称更新

全局替换 "AI Revenue Management OS" → "AI Revenue Management OS"。涉及：
- `package.json` 的 name/description
- 页面标题和 metadata
- 导航组件中的产品名
- 文档引用

## 4. 应该调整的部分（对齐）

**调整**：确认"客户"入口是否保留。根据规格，`/customers` 不在用户直接使用的页面中，而是作为下钻页面。建议从主导航移除，改为从对话中跳入。

最终导航应为：

| 导航项 | 路由 | 说明 |
|--------|------|------|

| 商机管道 | `/pipeline` | Pipeline 看板 |

### 4.3 Task Card 系统简化

## 6. 核心原则（实施时必须遵守）

1. **不新增页面**，只做减法和增强
2. **对话是唯一主入口**，所有功能通过对话触达
3. **卡片标注 Agent 来源**，让用户知道是谁在说话
4. **保持 suggestion → confirm → apply → sync 分离**
5. **每张卡片必须有证据可追溯**
6. **mock-first**：所有新增内容优先用 seed/mock 数据体现
