# Agent Panel IA Redesign

日期：2026-03-09

状态：待评审

范围：

- 只重构右侧 Agent 面板的信息架构与文案层级
- 不改动业务对象模型
- 不改动三个角色工作台的顶层框架

相关依据：

- `AGENTS.md`
- `/Users/wayne/.codex/skills/frontend-design/SKILL.md`

## 1. 问题定义

当前右侧 Agent 面板已经具备：

- 可收起 / 展开
- 输入提问
- 拖拽决策卡作为上下文
- 带页面 context 请求模型

但它的信息架构仍然过于冗余。

当前问题主要有四类：

### 1.1 同一件事被重复说了 3 次以上

当前面板中，用户会连续看到：

- 页面标题
- 页面描述
- 当前页上下文卡
- 对话输入说明文案
- 建议提示词

这些内容本质都在重复同一个信息：

```text
你现在在这个页面里，可以围绕当前任务继续问。
```

这会造成：

- 阅读负担高
- 真正重要的信息被淹没
- 输入区和附件区失去视觉主角地位

### 1.2 把“给模型看的上下文”错误地全部展示给了用户

当前设计把页面 context 几乎原样显示出来。

但这里有一个根本原则：

```text
模型需要很多上下文，不代表用户也需要看到这些上下文。
```

用户需要的是：

- 当前正在围绕什么任务协作

用户不需要反复阅读：

- 系统如何理解当前页面
- 冗长的内部上下文拼接结果

### 1.3 输入区周围的说明过多

当前输入区附近同时出现：

- “对话输入”标题
- 一段帮助文案
- 附件说明
- placeholder
- 推荐提问

这会导致用户在准备提问前，先被迫读一堆说明。

正确方向应该是：

- 让输入框本身成为主动作区
- 说明尽量内嵌到 placeholder 或轻量提示中

### 1.4 推荐提问过长，且重复当前上下文

当前推荐提问有两个问题：

1. 文案太长
2. 经常把当前决策卡标题重新说一遍

这会让推荐提问看起来像“系统又解释了一次”，而不是“给你一个下一步问题按钮”。

## 2. 设计目标

新的 Agent 面板应当更像：

- 任务对话工作区

而不是：

- 冗长说明栏

用户打开面板后应该只快速理解三件事：

1. 我现在围绕什么任务协作
2. 我附加了哪些上下文
3. 我现在可以直接问什么

---

## 3. 核心原则

### 3.1 一个层级只回答一个问题

顶部：

- 当前任务是什么

中部：

- 我们已经聊了什么

底部：

- 我现在要输入什么

不要让一个区块同时承担：

- 说明
- 教学
- 上下文展示
- 推荐问题

### 3.2 上下文默认隐藏，任务摘要前置

页面 context 不应以一大段解释卡出现。

应当改成：

- 一条短任务摘要

例如：

- `当前任务：先判断是否该直接推进方案澄清`

如果需要更详细的上下文，应通过：

- 已附加卡片
- 对话历史
- 模型内部 context

来承载，而不是默认全部显示。

### 3.3 输入动作是主视觉焦点

用户来这里最主要的动作不是读说明，而是：

- 输入问题
- 发送
- 围绕附件继续追问

因此视觉优先级应是：

1. 输入框
2. 附件条
3. 最近消息
4. 当前任务

而不是现在这样把说明文案放在输入区上面。

### 3.4 推荐问题必须短

推荐问题应该像快捷动作，不应该像二次解释。

推荐形式建议：

- `为什么还没到商机阶段？`
- `缺什么信息？`
- `今天先做哪一步？`

不要使用：

- 围绕“某某项目今天应直接推进方案澄清”我应该先问什么？

这种文案太长，也在重复页面信息。

### 3.5 附件只显示“我带了什么”

附件区的目标不是解释卡片内容，而是告诉用户：

- 当前对话附加了哪些对象

因此附件只需展示：

- 类型
- 标题
- 删除按钮

不需要再写一层“决策卡上下文”的完整说明块。

---

## 4. 新的信息架构

建议把右侧 Agent 面板改为五层：

1. Header
2. Task Strip
3. Message Thread
4. Attachment Strip
5. Composer

### 4.1 Header

内容：

- 面板标题
- 当前角色/页面标识
- 收起按钮

示例：

- `一线销售作战室`
- `作战对话`

不再需要：

- 大段页面描述

### 4.2 Task Strip

目标：

- 只用一句话告诉用户当前正在围绕什么任务协作

内容建议：

- `当前任务`
- 一句短摘要

示例：

- `当前任务：先判断是否应直接推进方案澄清`

不要再出现：

- 多句背景介绍
- “不要把首页做成个人 CRM 面板”这类产品定义文案

这些适合在设计文档里，不适合在工作面板里反复展示。

### 4.3 Message Thread

目标：

- 只显示对话内容

空状态时只保留一句：

- `输入问题或拖入卡片开始细聊`

不要在这里堆教学文案。

### 4.4 Attachment Strip

目标：

- 显示已附加的卡片/对象

每个附件形式建议：

- 小型 chip / pill
- 图标或类型
- 标题
- 删除

示例：

- `决策卡 · 曜石游戏新作买量试用项目`

不要用现在这种大块附件卡片。

### 4.5 Composer

结构建议：

- 输入框
- 内嵌发送图标按钮
- 最多 2 个快捷问题

输入框上方不要再加说明段落。

说明仅通过：

- placeholder

表达即可。

placeholder 示例：

- `问一个具体问题，或继续围绕已附加卡片深聊`

发送按钮应当做得很轻：

- 放在输入框内部右下角或右侧内嵌位置
- 使用小飞机或箭头图标
- 视觉上是输入动作的一部分

不建议：

- 单独占据一整行
- 做成大型主按钮
- 和输入框形成两个竞争焦点

快捷问题必须收敛到最多 2 个，并且短：

- `缺什么信息？`
- `下一步做什么？`

当用户已经输入内容时，快捷问题应弱化或隐藏。

---

## 5. 建议删除项

以下内容建议直接删除：

1. Header 下的大段页面说明
2. 当前页上下文的大段说明卡
3. 输入框上方的说明段落
4. 过长的推荐提问
5. 大块“决策卡上下文”展示卡

这些信息不是完全没用，而是：

- 不应默认全部展示
- 应转入模型内部 context 或更轻量的 UI 形式

---

## 6. 建议保留项

以下能力应保留，但 UI 形式应压缩：

1. 收起 / 展开
2. 页面上下文感知
3. 拖拽卡片进入对话
4. 附件可移除
5. 推荐问题
6. 消息线程

---

## 7. 建议文案规范

### Header

保留：

- `一线销售作战室`

新增轻量副标题：

- `作战对话`

### Task Strip

格式：

- `当前任务：<一句话>`

例如：

- `当前任务：先判断是否应直接推进方案澄清`

### 输入框 placeholder

示例：

- `问一个具体问题，或拖入卡片继续细聊`

### 快捷问题

示例：

- `为什么还没到商机阶段？`
- `缺什么信息？`

## 9. 结论

这次问题不是“文字多一点”，而是：

```text
当前 Agent 面板把系统内部上下文展示逻辑，
错误地做成了用户界面的主结构。
```

新的 Agent 面板应当回到一个更清晰的工作区模型：

- 顶部只说当前任务
- 中部只保留对话
- 底部只保留附件与输入

一句话总结：

```text
把“解释界面”改成“任务对话界面”，
把“上下文展示”改成“上下文附带”。
```
