# Current Documentation

这是当前仓库唯一应该优先阅读的文档入口。

## 阅读顺序

1. `AGENTS.md`
2. `docs/current/product-direction.md`
3. `docs/current/implementation-status.md`
4. `APIkey.md`

## 这套 current 文档回答什么

- 当前产品主线到底是什么
- 当前 `main` 已经实现到哪里
- 哪些旧文档已经归档，不应再当成 live spec
- 历史里有哪些经验必须保留成护栏

## 当前统一结论

当前 canonical 方向统一为：

- `meeting-first` 作为业务语义基座
- `account thread` 作为持续上下文
- `deal` 作为经营投影
- `raw intake` 作为上游输入链路
- `conversational agent os` + `task cards` 作为当前主交互探索
- 对象页、详情页和支撑页继续作为证据与执行层存在

## 文档分层

- `docs/current/`：当前唯一 canonical 文档
- `docs/insights/`：时间线、洞察、反回退规则
- `docs/history/`：历史设计稿、实现计划、旧阶段产物
- `docs/plans/`：仅存放当前进行中的设计稿和实施计划
